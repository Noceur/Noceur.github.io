Case study

Homeworld Mobile — tutorial & narrative missions

Problem. A mobile entry in the Homeworld universe needed a tutorial path and a cinematic narrative arc that played cleanly off the standalone systems for combat, progression and unlocks. Onboarding had to teach without lecturing; narrative missions had to trigger reliably from a player’s state at any point in their progression.

Approach. I was the sole level designer on the project and focused on the tutorial and the narrative-mission chain. All mission logic was authored in an in-house node-based tool similar in spirit to Unreal Blueprints — nodes for triggers, fleet spawns, camera moves, narrative beats, and unlock gates — with the unlock-gate nodes wiring missions directly into the global progression graph.

Result. Full tutorial path plus a chain of narrative missions, all connected to player progression. A playthrough of the story content is available on YouTube.

Dawn of Ages — Maya batch rig exporter

Problem. An animator on Dawn of Ages was burning through hours repeating the same FBX export setup for every rig change. The action list per export was the same every time: strip out the control rig, strip out specific bones, push the studio’s FBX settings, export, then put the user’s settings back — multiplied by however many rigs needed re-exporting that day.

Approach. A Maya tool in Python with JSON-defined profiles. Each profile stores its own “objects to keep,” “objects to delete” and target path, so a single button runs the full sequence per rig, or all of them in batch. The tool wraps every export in an undo chunk and pushes/pops the user’s personal FBX settings around the studio-forced ones, so nothing in the artist’s session is mutated when the run finishes.

Tool interface — profile setup at the top, per-profile actions below.

Usage cheatsheet

Profile name — the label that later appears in the dropdown.
Add objects to save — selected objects (and their children) added to the per-profile export list. Additive: clicking again appends.
Add objects to delete — objects excluded from the export, typically the control rig or specific bones.
Profile save location — target FBX path.
Wipe new profile data — clear the in-progress save/delete lists.

Then per existing profile:
Export profile objects runs one profile end-to-end:

  • Open an undo chunk so the scene can be restored after.
  • Select the kept objects (with hierarchy).
  • Delete the unwanted objects.
  • Push the user’s FBX settings, set the studio/tool settings.
  • Export the FBX.
  • Pop the user’s settings back and undo the scene mutations.

Export all profile objects — the same loop across every profile in the active JSON.
Select profile objects — selects the kept mesh set without exporting, useful for debugging a profile that mis-fires.
Delete profile — self-explanatory.

Settings actions save or load the whole profile collection to a JSON, so artists can share their export configs.

        profileName = cmds.optionMenu(self.profileMenu, query = True, value = True)
        self.activeProfile = self.settings['profiles'][profileName]


    def selectObjs(self, *args):
        if self.activeProfile:
            mySel = cmds.ls(self.activeProfile['objs'])
            cmds.select(mySel, hi=True)
        else:
            print ("no active profile")


    def saveProfile(self, *args):
        # Perform checks
        profile = cmds.textFieldGrp(self.profileName, query=True, text=True)
        if not self.saveProfileCheck(profile):
            return

        fPath = self.getSaveProfileSaveLocation(profile)

        if fPath:
            self.settings['profiles'][profile] = {}
            self.settings['profiles'][profile]['objs']    = self.saveObjs
            self.settings['profiles'][profile]['delObjs'] = self.delObjs
            self.settings['profiles'][profile]['path']    = fPath
            cmds.menuItem(label=profile, parent=self.profileMenu)
            self.setActiveProfile()

        if self.wipeProfileAfterSave:
            self.wipeState()


def undoChunkOpen(open=True):
    # opens and closes a undo section
    if not open:
        cmds.undoInfo(chunkName="state", closeChunk=True)
        return
    cmds.undoInfo(chunkName="state", openChunk=True)


def saveJson(path, data):
    with open(path, 'w') as outfile:
        json.dump(data, outfile)


def loadJson(path):
    with open(path, "r") as json_file:
        return json.load(json_file)


def getFBXSettings():
    mel.eval('FBXPushSettings;')


def setFBXSettings():
    mel.eval('FBXPopSettings;')


def exportFBX(exportFileName, min_time=False, max_time=False):
    getFBXSettings()
    # Geometry
    mel.eval("FBXExportSmoothingGroups -v true")
    mel.eval("FBXExportHardEdges -v false")
    mel.eval("FBXExportTangents -v false")
    mel.eval("FBXExportSmoothMesh -v true")
    mel.eval("FBXExportInstances -v false")
    mel.eval("FBXExportReferencedAssetsContent -v false")
    mel.eval("FBXExportAnimationOnly -v false")
    mel.eval("FBXExportBakeComplexAnimation -v true")
    if not min_time == False:
        mel.eval("FBXExportBakeComplexStart -v " + str(min_time))
    if not max_time == False:
        mel.eval("FBXExportBakeComplexEnd -v " + str(max_time))
    mel.eval("FBXExportBakeComplexStep -v 1")
    mel.eval("FBXExportUseSceneName -v false")
    mel.eval("FBXExportQuaternion -v euler")
    mel.eval("FBXExportShapes -v true")
    mel.eval("FBXExportSkins -v true")
    mel.eval("FBXExportConstraints -v false")
    mel.eval("FBXExportCameras -v false")
    mel.eval("FBXExportLights -v false")
    mel.eval("FBXExportEmbeddedTextures -v false")
    mel.eval("FBXExportInputConnections -v false")
    mel.eval("FBXExportUpAxis y")
    mel.eval("FBXExportFileVersion -v FBX201600")
    mel.eval("FBXExportInAscii -v true")
    cmds.file(exportFileName, exportSelected=True, type="FBX export", force=True, prompt=False)
    setFBXSettings()
